# 文献阅读与讲义生成工作流 - 设计文档

## 概述

创建一个模拟用户阅读文献、学习知识、梳理教材、生成个性化 LaTeX 讲义的工作流系统。

## 核心需求

### 1. PDF 输入与解析
- 支持 PDF 格式的教材、论文作为输入
- 提取书籍的章节结构（章、节、小节）
- 可同时处理单本或多本 PDF

### 2. 两阶段阅读流程

**阶段一：扫描阶段**
- 快速了解内容概览
- 与用户已有知识建立关联
- 识别需要先备的知识

**阶段二：逐章阅读**
- 读每章前先展示核心目标（如何从引理推导主定理）
- 区分核心内容 vs. 细节铺垫
- 区分简单练习 vs. 复杂但深刻的内容
- 遇到不懂的地方可以随时提问

### 3. 交互式学习
- 用户可以随时提问："我不懂这个定理"
- Claude 知道具体是哪个定理/位置
- 对话回答：让初学者能看懂
- 讲义记录：Stein 风格的 motivation 写作，有流畅叙事

### 4. LaTeX 讲义生成
- **结构**：每章/每节独立 .tex 文件，main.tex 统一整合
- **风格**：详细注解型 - 在定义、命题、证明之间穿插用户的理解、疑问、解释
- **跨文件引用**：不同章节之间可交叉引用
- **写作风格**：模仿 Stein《复分析》《傅里叶分析》的 motivation 风格

### 5. 阅读记忆系统
- 记住用户读过的章节和位置
- 记录提问历史（哪些定理不懂）
- 记录跳过/困惑的地方
- 学习用户偏好的写作风格
- 跨会话记忆

### 6. 进度仪表盘
- 多种格式展示阅读进度：
  - Markdown 文档（可视化）
  - JSON 数据文件
  - Mermaid 图

## 系统架构

### 组件

| 组件 | 功能 |
|------|------|
| PDF 解析模块 | 读取 PDF，提取章节结构 |
| 阅读记忆系统 | 跨会话记住习惯、问题、偏好 |
| 对话引擎 | 实时问答，理解上下文位置 |
| LaTeX 生成器 | 按 Stein 风格生成带注解讲义 |
| 进度仪表盘 | 显示阅读状态（MD + JSON） |

### 数据流

```
输入 PDF → 扫描阶段 → 章节概览 → 逐章阅读 → 实时问答 → 更新讲义
                ↓
           生成进度仪表盘
```

### 讲义目录结构

```
project/
├── main.tex
├── chapters/
│   ├── chapter1/
│   │   ├── section1.tex
│   │   ├── section2.tex
│   │   └── ...
│   ├── chapter2/
│   │   └── ...
│   └── ...
├── appendix/
│   └── qa.tex (用户的问答记录)
└── assets/
```

### 记忆存储

用户偏好存储在 `~/.claude/user-preferences.json`，包含：
- 写作风格偏好
- 阅读习惯
- 提问模式

## 交互方式

1. 用户告诉 Claude 要读哪章哪节
2. Claude 展示章节概览和核心目标
3. 用户阅读过程中随时提问
4. Claude 实时回答并更新对应 .tex 文件
5. 用户可以讨论读后感，Claude 记录到记忆系统
